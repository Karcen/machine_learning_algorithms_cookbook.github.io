Decision Tree

Supervised Learning - Classification/Regression Algorithm

Classification Regression Tree-based Interpretable

Algorithm Overview

Decision Tree is a tree-like model of decisions and their possible consequences. It uses a tree structure where each internal node represents a test on an attribute, each branch represents the outcome of the test, and each leaf node represents a class label or a regression value.

Key Characteristics

  • Tree-structured decision model
  • Handles both classification and regression
  • Highly interpretable (white-box model)
  • Requires little data preprocessing
  • Prone to overfitting without pruning

Applications

  • Medical diagnosis
  • Credit risk assessment
  • Customer segmentation
  • Fraud detection
  • Manufacturing quality control

2. Splitting Criteria

Gini Impurity

\[ G = 1 - \sum_{i=1}^{c} p_i^2 \]

Measures probability of misclassifying a randomly chosen element

Entropy

\[ H = -\sum_{i=1}^{c} p_i \log_2 p_i \]

Measures uncertainty or randomness in the data

Information Gain

\[ IG = H_{\text{parent}} - \sum_{i=1}^{k} \frac{n_i}{n} H_{\text{child}_i} \]

Reduction in entropy after splitting

Variance Reduction (Regression)

\[ VR = \text{Var}(Y) - \sum_{i=1}^{k} \frac{n_i}{n} \text{Var}(Y_i) \]

Used for regression trees

3. Pruning Techniques

Cost-Complexity Pruning

\[ R_\alpha(T) = R(T) + \alpha |T| \]

\( R(T) \) is the tree's error rate, \( |T| \) is the number of leaves, and \( \alpha \) controls the trade-off between complexity and accuracy

4. Algorithm Variants

ID3

Uses Information Gain, handles categorical features

C4.5

Uses Gain Ratio, handles continuous features

CART

Uses Gini Impurity, produces binary trees

Implementation

import numpy as np
from sklearn.tree import DecisionTreeClassifier, DecisionTreeRegressor, export_graphviz
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score, mean_squared_error
from sklearn.datasets import load_iris, load_boston
import matplotlib.pyplot as plt
from sklearn import tree

# Classification example with Iris dataset
iris = load_iris()
X_clf, y_clf = iris.data, iris.target
X_train_clf, X_test_clf, y_train_clf, y_test_clf = train_test_split(
    X_clf, y_clf, test_size=0.3, random_state=42
)

# Create and train classification tree
clf_tree = DecisionTreeClassifier(
    criterion='gini',
    max_depth=3,
    min_samples_split=2,
    min_samples_leaf=1,
    random_state=42
)
clf_tree.fit(X_train_clf, y_train_clf)

# Make predictions
y_pred_clf = clf_tree.predict(X_test_clf)
accuracy = accuracy_score(y_test_clf, y_pred_clf)
print(f"Classification Accuracy: {accuracy:.2f}")

# Regression example with Boston housing data
boston = load_boston()
X_reg, y_reg = boston.data, boston.target
X_train_reg, X_test_reg, y_train_reg, y_test_reg = train_test_split(
    X_reg, y_reg, test_size=0.3, random_state=42
)

# Create and train regression tree
reg_tree = DecisionTreeRegressor(
    criterion='squared_error',
    max_depth=5,
    min_samples_split=2,
    min_samples_leaf=1,
    random_state=42
)
reg_tree.fit(X_train_reg, y_train_reg)

# Make predictions
y_pred_reg = reg_tree.predict(X_test_reg)
mse = mean_squared_error(y_test_reg, y_pred_reg)
print(f"Regression MSE: {mse:.2f}")

# Visualize the tree
plt.figure(figsize=(12, 8))
tree.plot_tree(clf_tree, filled=True, feature_names=iris.feature_names, class_names=iris.target_names)
plt.show()

# Feature importance
print("\nFeature Importance (Classification):")
for name, importance in zip(iris.feature_names, clf_tree.feature_importances_):
    print(f"{name}: {importance:.4f}")

print("\nFeature Importance (Regression):")
for name, importance in zip(boston.feature_names, reg_tree.feature_importances_):
    print(f"{name}: {importance:.4f}")

Mathematical Theory

Computational Complexity

Training time complexity is \( O(n \log n) \) for sorting features, and \( O(n d \log n) \) for finding optimal splits, where n is the number of samples and d is the number of features.

Statistical Properties

Decision trees are non-parametric models with high variance and low bias. They can represent any boolean function given sufficient depth, making them universal approximators.

Bias-Variance Tradeoff

High Bias: Shallow trees with limited depth
High Variance: Deep trees that overfit to training data
Optimal: Balanced tree depth with pruning

Information Theory Foundations

Information gain is based on Claude Shannon's information theory. The entropy function \( H(X) = -\sum p_i \log p_i \) measures the uncertainty in a random variable.

Consistency Properties

Under mild conditions, decision trees are consistent estimators - as the sample size increases, the tree's prediction error converges to the Bayes error rate.
